# Original AWS Architecture — FlashMQ → Telegraf → InfluxDB2

This directory preserves the **original data pipeline** for reference and comparison.
The new architecture (NATS JetStream + Parquet/S3) was designed to replace this at ~95% lower cost.

---

## Architecture

```
IoT Sites (MQTT devices)
        │
        ▼
  ┌─────────────┐
  │  FlashMQ    │  High-performance MQTT broker (C++, AGPL v3)
  │  port 1883  │  Handles thousands of concurrent publishers
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │  Telegraf   │  InfluxData metrics collection agent
  │  MQTT input │  Subscribes to batteries/#, parses JSON
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │  InfluxDB2  │  Time-series database
  │  port 8086  │  Flux query language, retention policies
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │   Grafana   │  Dashboards + alerting
  │  port 3000  │
  └─────────────┘
```

## Monthly AWS Cost Breakdown (Baseline)

| Service | Cost/month | Driver |
|---|---|---|
| EC2 (InfluxDB2 + FlashMQ) | $36,000 | Large instances + InfluxDB memory requirements |
| EFS (InfluxDB2 data) | $10,000 | Time-series data stored on Elastic File System |
| IoT Core | $3,000 | MQTT message routing rules engine |
| CloudWatch | $2,600 | High-resolution custom metrics, log ingestion |
| Lambda | $2,000 | Rules engine triggers, transformations |
| S3 | $3,000 | Backups, exports |
| Data Transfer | $3,600 | Egress to clients |
| **Total** | **~$76,200** | |

### Why EC2 is so expensive
InfluxDB2 is memory-hungry. A production deployment handling thousands of series
at 1s resolution typically requires `r5.2xlarge` or larger (~$0.504/hr = ~$368/mo per instance).
Multiple instances for HA, plus the FlashMQ broker instances, quickly reach $36k/mo.

### Why EFS is expensive
InfluxDB2 uses its own storage engine (TSM/IOx). Mounting EFS for InfluxDB2 data gives
shared storage across AZ but costs ~$0.30/GB-month. At the data volumes generated by
thousands of battery cells at 1s resolution, 30-day retention easily reaches 30+ TB.

## New Architecture Comparison

| Aspect | Original (this dir) | New (NATS + Parquet) |
|---|---|---|
| Cost/month | ~$76,200 | ~$500–2,000 |
| Savings | — | ~$74,000/mo (~$888k/yr) |
| Storage engine | InfluxDB2 TSM on EFS | Parquet on S3 |
| Message broker | FlashMQ (MQTT) | FlashMQ → NATS JetStream |
| Data loss risk | Telegraf buffer loss | Zero (ack-after-upload) |
| Historical queries | Flux over InfluxDB | DuckDB over S3 Parquet |
| Schema flexibility | Measurement + tags | Any columnar schema |
| Retention | Configurable in InfluxDB | S3 lifecycle policies |

## Directory Structure

```
orig_aws/
├── README.md                    ← this file
├── docker-compose.yaml          ← local dev (all services)
├── setup_aws.sh                 ← native EC2 install (no Docker)
├── requirements.txt             ← Python deps for generator
├── flashmq/
│   └── flashmq.conf             ← FlashMQ broker config
├── telegraf/
│   └── telegraf.conf            ← MQTT consumer → InfluxDB2
├── influxdb/
│   └── README.md                ← InfluxDB2 setup, Flux queries
├── grafana/
│   └── provisioning/
│       └── datasources/
│           └── influxdb.yaml    ← Auto-provision datasource
└── source/
    ├── config.yaml              ← Generator config
    └── generator.py             ← MQTT battery data simulator
```

## Quick Start (Docker Compose — local dev)

```bash
cd orig_aws
docker compose up -d

# Wait ~30s for InfluxDB2 to initialize, then:
# InfluxDB2 UI:  http://localhost:8086   (admin / adminpassword123)
# Grafana:       http://localhost:3000   (admin / admin)

# Start the data generator
pip install -r requirements.txt
python source/generator.py --config source/config.yaml

# Data should appear in InfluxDB2 under bucket: battery-data
```

---

## Quick Start (Native — AWS EC2)

```bash
chmod +x setup_aws.sh && sudo ./setup_aws.sh
python source/generator.py --config source/config.yaml
```
